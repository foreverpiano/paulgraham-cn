Viaweb 常见问题

阅读原文

编辑器如何处理客户端会话？

每个用户对应一个独立的 Lisp 进程。当有人登录编辑自己的网站时，我们会启动一个新的进程，并将他们全部数据加载到内存中。从那一刻起，他们就与这个进程保持着持续的对话。

由于所有数据都已经加载到内存，我们无需从磁盘读取任何内容，也不用为响应 HTTP 请求而启动进程。我们只需对内存中保存的一个闭包求值即可。

你们用什么来做 HTTP 服务器？

最初编辑器有自己的 HTTP 服务器，由 Robert Morris 用 Common Lisp 编写。后来我们改用他修改过的 Apache 版本，让它能与 Lisp 通信。

你们用的是哪种 Lisp？Clisp。

你们是否使用真正的 continuation 来保存状态？

没有，我们在 Common Lisp 中用宏来模拟 continuation，正如 论 Lisp 一书中所描述的那样。

你们用什么数据库？

我们没有使用数据库，只是把所有数据都存在文件里。Unix 文件系统很擅长保护数据不丢失，尤其是把文件放在 NetApp 上的时候。

很多人错误地认为基于 Web 的应用就是数据库的界面。桌面应用显然不只是数据库的界面，为什么基于 Web 的应用就该不同呢？难点不在于数据存在哪里，而在于软件到底要做什么。

在我们做 Viaweb 期间，那些伪技术人士——比如风投和行业分析师——因为我们没用数据库、只用廉价的 Intel 机器跑 FreeBSD 做服务器，没少给我们施压。但后来我们被 Yahoo 收购时发现，他们也只是把所有东西存在文件里，而且他们的服务器同样是廉价的 Intel 机器，跑的也是 FreeBSD。

（在互联网泡沫时期，Oracle 曾经打广告说 Yahoo 运行在 Oracle 软件上。我觉得这难以置信，就四处打听。后来发现是 Yahoo 的会计部门在使用 Oracle。）

你的联合创始人是不是写过那个著名蠕虫、现在在 MIT 任教授的 Robert Morris？

是的。

你们的种子资金从哪里来的？

我们从几位私人投资者那里拿到钱，也就是业内所说的“天使投资人”。我们的投资人相当认真，几乎可以算是 VC，但他们并不是真正有品牌的大型风险投资公司。

我们做 Viaweb 的成本非常低，总共只花了约 200 万美元。被收购时我们已经接近盈亏平衡，所以本来也不会再花太多钱。

“Viaweb”该怎么发音？

官方说法是两种发音都可以：vee-a-web 或者 vie-a-web。我们自己都用前一种，但其他人（包括 Yahoo 的人）似乎更喜欢后一种。

如果重来一次，你会做什么不同的事？

技术上不会有太大改变。我觉得我们当时最应该做、却没有做的一件事，就是自己开一家在线商店。我们用这个编辑器给自己做网站，所以有很强的动力把它做好。但对于电子商务那部分功能，我们只能间接理解。
