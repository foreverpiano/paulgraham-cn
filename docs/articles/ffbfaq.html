FFB 常见问题

阅读原文

什么是 FFB？

能反击的过滤器（反击过滤器）。

FFB 会做什么？

对每封新收到的垃圾邮件中的所有 URL 进行爬取，只要该服务器的 IP 地址在黑名单上。理想情况下，HTTP 请求会分散在垃圾邮件的整个生命周期内——也就是“客户”会对其作出回应的那段时间。我猜测大多数回应发生在垃圾邮件发出后的几个小时内。夜间发送的垃圾邮件情况可能有所不同。

垃圾邮件发送者能否搭建足够快的服务器来应对 HTTP 请求洪流？

那些 hardcore 的垃圾邮件发送者可能可以，但像 AmericanSingles.Com 这样的网站就不行了。

FFB 会产生什么样的带宽成本？

如果有 10 万个 FFB，垃圾邮件发送者的网站总内容为 100k，每个 FFB 针对每封垃圾邮件爬取网站 10 次，这将产生 100 GB 的流量传输。按照目前垃圾邮件托管的价格，这大约会让垃圾邮件发送者每封垃圾邮件花费 1000 到 4000 美元。

入门级的垃圾邮件托管方案每月流量限制远低于 100 GB，因此 FFB 能在“客户”有机会回应之前就让这类网站关闭。在有 FFB 的世界里，想成为垃圾邮件发送者会非常困难。

垃圾邮件发送者难道不能保护自己免受爬虫攻击吗？

他们或许可以，通过在页面中加入隐形链接。任何跟随这类链接的必定是爬虫，可以安全地忽略。但对抗 FFB 最简单的方法是加入有效的退订链接。这对刚起步的小型垃圾邮件发送者和雇用垃圾邮件发送者来获取流量的“合法”网站尤其必要。这些网站既没有基础设施，也没有专业知识来防范 FFB。

获取网页信标（web beacon）难道不会暴露你的地址是活跃的吗？

是的，这可能会招致更多垃圾邮件。但它也会让网页信标不再能有效衡量邮件打开率。而且你同时也会点击退订链接，这会让退订链接在 FFB 的推动下变得更加普遍。

这难道不会鼓励人们发送推广按点击付费网站的垃圾邮件吗？

如果还有网站按点击付费，它们肯定已经有一些方法来保护自己免受通过垃圾邮件产生流量的人的侵害。所以这样做不会有钱可赚。

这是个坏主意，因为它只会消耗更多带宽。

这就好比说我们不应该有警察，因为除了犯罪造成的损失之外，我们还要让人们放下生产性工作去追捕罪犯。如果 FFB 能让有效的退订链接变得普遍，最终结果是带宽的净消耗减少。

我并不是建议拨号上网的用户使用 FFB，而是那些有多余带宽的用户——大学和公司里的用户，以及使用 DSL 线路的用户。

对垃圾邮件发送者发动拒绝服务攻击难道不违法吗？

这种攻击将是 uncoordinated（非协同）的，而且分布得非常广，单个客户端不需要发送太多 HTTP 请求。我不是建议某个过滤器对垃圾邮件发送者的网站发起 1 万个 HTTP 请求，而最多是几百个。

很难有人能指控你发送几百个 HTTP 请求就是在对他发动 DoS 攻击。如果几百个请求就算 DoS 攻击，那我已经好几次对 eBay 犯下了 DoS 的罪行。

无论如何，垃圾邮件发送者喜欢保持低调。我无法想象他们——尤其是那些在海外的——会试图寻求法律保护。他们很可能已经遭受过传统的 DoS 攻击，但我没听说他们中的任何人求助于法律。

这可能会被用来对无辜受害者发动 DoS 攻击。

这正是需要黑名单的原因。一个网站不会仅仅因为在垃圾邮件中被提及就遭到攻击。它必须同时在垃圾邮件中被提及且出现在黑名单上。

黑名单该如何管理？

我没有执着于某一种做法。但我怀疑这无法完全自动化。必须由人工检查网站，以保护无辜网站不被列入黑名单。

如果 FFB 对垃圾邮件发送者构成威胁，他们就会故意让 FFB 攻击无辜受害者来败坏其名声。因此黑名单应该同时包含 IP 地址和域名。否则垃圾邮件发送者可以修改黑名单网站的 DNS 记录，让它指向无辜受害者。

任何运行黑名单的人都应该默认假设垃圾邮件中出现的任何 URL 都是 Joe job（嫁祸攻击）的受害者，只有在人工检查后确认并非如此时才将其列入黑名单。

黑名单难道不会被滥用吗？

会。老实说，这是这个方案的薄弱环节。如果一个被广泛使用的 FFB 黑名单由 MAPS 之类的疯子运营，可能会造成很大损害。而且所有黑名单似乎最终都会变得 黑名单的宿命（腐败）。

FFB 黑名单比邮件服务器黑名单危险性小，因为仅被列入黑名单本身不足以给网站带来麻烦。只有当你已经在黑名单上，并且收到一封包含你 URL 的新垃圾邮件时，才会被爬取。另一方面，FFB 将由个人用户运行，他们对所订阅的黑名单不会像 ISP 那样有辨别力。

垃圾邮件发送者难道不能起诉或 DoS 黑名单吗？

为防范 DoS 攻击，黑名单可能需要通过 P2P 网络来分发。

我认为诉讼不会构成太大威胁。一个列出被垃圾邮件宣传的网站的黑名单，可以合理声称它并非专门供 FFB 使用。这种黑名单在垃圾邮件过滤中确实有真实需求。为了规避过滤器，垃圾邮件发送者现在会定期更换域名。当过滤器第一次看到某个域名时，如果有一个列表可以查阅，看看其他用户是否报告过包含该域名的垃圾邮件，那将非常有帮助。

如果 FFB 碰巧也使用了这个列表，那只是管理员不知情的情况下发生的，就像那些淘气的“联盟营销者”在网站不知情的情况下发送垃圾邮件一样。

这难道不会漏掉使用 Migmaf 式代理的垃圾邮件发送者吗？

会。但如果我们迫使垃圾邮件发送者采取这种措施来求生，我想我们会把他们中的很多人逼出这一行。刑事司法系统对编写病毒的重视程度远高于伪造邮件头。我认为目前很多垃圾邮件发送者不会愿意冒这个险。

如果垃圾邮件发送者入侵他人网站服务器并用其进行重定向呢？

这对他们没有帮助。FFB 仍然会干扰垃圾邮件发送者的“客户”。

如果垃圾邮件发送者使用 Geocities 等网站的跳转呢？

这类垃圾邮件非常少见，可能是因为 Geocities 已经想出办法阻止垃圾邮件发送者这样利用他们。没有掌握这一点的托管服务很快就会吸取教训。

为什么要把邮件纳入这个系统？为什么不直接建立一个垃圾网站黑名单并鼓励大家去攻击它们？

有几个人写信建议开展这类“DDoS@Home”项目。（两位不愿透露姓名的通讯者同时发明了这个吸引人的名字。）但我认为邮件应该保留在系统中，原因有两个：(a) 它告诉你该攻击哪些网站，以及何时攻击；(b) 如果将其作为过滤器的一部分，你能吸引更多用户。

另一方面，如果某个团体真的成功发起针对垃圾邮件发送者的 DDoS@Home 项目，那将会非常有趣。我会报名参加。
