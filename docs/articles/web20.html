Web 2.0

November 2005 · 阅读原文

2005 年 11 月

“Web 2.0”这个词到底有没有意义？直到不久前我还认为它毫无意义，但实际情况比我想象的更复杂。最初，它确实毫无意义。现在它似乎已经获得了一层含义。然而，那些讨厌这个词的人可能还是对的，因为如果它真的指代我所理解的那种东西，我们其实并不需要这个词。

我第一次听到“Web 2.0”这个说法，是在 2004 年那场以 Web 2.0 为名的会议上。当时它的含义据说是“把网络当作平台”，我理解为基于网页的应用。[[1]]

所以今年夏天在一次会议上，当蒂姆·奥莱利（Tim O'Reilly）主持一场试图给“Web 2.0”下定义的讨论时，我感到很惊讶。它不是已经意味着把网络当作平台吗？如果它原本并没有明确含义，那我们又为什么需要这个说法呢？

起源

蒂姆说，“Web 2.0”这个词最早出现在“O'Reilly 和 Medialive International 之间的一次头脑风暴会议”中。Medialive International 是什么？根据他们的网站，是“技术贸易展和会议的举办方”。所以那次头脑风暴大概就是为了这个。O'Reilly 想办一场关于网络的会议，他们在考虑应该叫什么名字。

我不认为当时有人故意暗示网络出现了一个新版本。他们只是想强调网络重新变得重要了。这是一种语义上的赤字支出：他们知道新事物即将到来，而“2.0”就用来指代那些不管最终是什么的新事物。

他们是对的。新事物确实来了。但新的版本号在短期内也带来了一些尴尬。在为第一次会议准备宣传的过程中，肯定有人觉得最好还是解释一下这个“2.0”到底指什么。不管它具体指什么，“把网络当作平台”至少不算太狭隘。

“Web 2.0 意味着把网络当作平台”这个说法在第一次会议之后就没怎么流传下来。到第二次会议时，“Web 2.0”似乎意味着某种关于民主的东西。至少，当人们在网上写到它时是这个意思。会议本身并不显得多么草根。它门票要 2800 美元，所以能去得起的只有风投和来自大公司的人。

然而奇怪的是，Ryan Singel 在 Wired News 上关于那场会议的文章却提到了“成群结队的极客”。当我一个朋友问 Ryan 时，他觉得这很意外。他说他 originally 写的是“成群结队的风投和商务开发人员”，后来简化为“成群结队”，结果编辑又把它扩展成了“成群结队的极客”。毕竟，一场 Web 2.0 会议理所当然应该挤满极客，对吧？

嗯，并非如此。现场大概只有 7 个极客。就连蒂姆·奥莱利都穿了西装，这景象太陌生，我一开始都没认出来。我看到他走过去，就对一个 O'Reilly 的人说：“那个人长得真像蒂姆。”

“哦，那就是蒂姆。他刚买了套西装。”我追上去，果然是他。他解释说这是他在泰国买的。

2005 年的 Web 2.0 会议让我想起互联网泡沫时期的贸易展，到处是四处游荡、寻找下一个热门初创公司的风投。那种奇怪的氛围来自一大群下定决心不想错过的人。错过什么呢？他们也不知道。不管即将发生什么——不管 Web 2.0 最终会是什么。

我不会因为风投又开始热衷投资就把它叫做“泡沫 2.0”。互联网确实是一件大事。那次泡沫破裂既是繁荣的过度反应，也是泡沫破裂的过度反应。可以预见的是，一旦我们开始从低谷中走出来，这个领域就会出现大量增长，就像大萧条前那些增长最迅猛的行业一样。泡沫所看准的地方

这次不会变成第二个泡沫的原因是 IPO 市场已经消失了。是由退出策略驱动的。他们在 90 年代末资助那么多可笑的初创公司，是因为希望把它们卖给容易上当的散户投资者；他们希望一路笑着进银行。现在这条路被堵死了。现在默认的退出策略是被收购，而收购方不像 IPO 投资者那样容易陷入非理性繁荣。你能看到的最接近泡沫估值的例子大概是鲁珀特·默多克（Rupert Murdoch）花 5.8 亿美元买下 Myspace。这也只偏离了大约 10 倍左右。如何为初创公司融资

1. Ajax

“Web 2.0”是否已经有了超出会议名称之外的含义？我不想承认，但它确实开始有了。当人们现在说“Web 2.0”时，我大概知道他们指什么。而我既鄙视这个词又能理解它，这一事实就是它已经开始有含义的最确凿证据。

它的含义中肯定包含 Ajax，这个词我到现在还是只能勉强不用引号就说出来。简单来说，“Ajax”的意思就是“JavaScript 终于能用了”。而这又意味着基于网页的应用现在可以做得更像桌面应用了。

在你读到这篇文章的时候，整整一代新的软件正在被编写出来，以利用 Ajax 的优势。自从微型计算机首次出现以来，还没有出现过如此大规模的新应用浪潮。就连微软都看到了这一点，但他们已经来不及做更多，只能泄露一些“内部”文档，试图给人一种他们引领这一新趋势的印象。

事实上，新一代软件的开发速度快得微软甚至都来不及引导，更别说自己内部开发了。他们现在唯一的希望是在谷歌之前把所有最好的 Ajax 初创公司都买下来。而这也会很难，因为谷歌在收购微型初创公司上的领先优势，和它几年前在搜索上的领先优势一样大。毕竟，典型的 Ajax 应用 Google Maps，就是他们收购的一家初创公司的成果。

所以具有讽刺意味的是，Web 2.0 会议最初的描述在一定程度上是正确的：基于网页的应用是 Web 2.0 的重要组成部分。但我相信他们只是偶然说对了。Ajax 的热潮直到 2005 年初 Google Maps 出现、“Ajax”这个词被创造出来之后才真正开始。

2. 民主

Web 2.0 的第二个重要元素是民主。我们现在已经有几个例子证明，当拥有正确的系统来引导他们的努力时，可以超越专业人士。Wikipedia 可能是最著名的。专家们对 Wikipedia 的评价一般，但他们忽略了关键点：它已经足够好了。而且它是免费的，这意味着人们真的会去阅读。在网络上，那些需要付费才能阅读的文章几乎等同于不存在。即使你愿意自己付费阅读，你也无法链接到它们。它们不属于这场对话的一部分。企业能从开源中学到什么

民主似乎获胜的另一个领域是决定什么算新闻。我现在除了 Reddit 之外几乎不看任何新闻网站。[[2]] 我知道如果发生了重大事件，或者有人写了一篇特别有趣的文章，它就会出现在那里。为什么还要费力去检查任何特定报纸或杂志的首页呢？Reddit 就像是整个网络的 RSS 订阅源，还自带质量过滤器。类似的网站包括 Digg——一个技术新闻网站，其受欢迎程度正迅速接近 Slashdot——以及 del.icio.us，那个开启了“标签”运动的协作书签网络。而 Wikipedia 的主要吸引力在于它足够好且免费，这些网站则表明，投票者比人工编辑做得明显更好。

Web 2.0 民主最戏剧性的例子不在于选择想法，而在于生产想法。我早就注意到，我在个人网站上读到的内容和我在报纸杂志上读到的内容一样好，甚至更好。现在我有了独立的证据：Reddit 上的热门链接通常指向个人网站，而不是杂志文章或新闻报道。

我为杂志撰稿的经历提供了一种解释。那就是编辑。他们控制你能写什么主题，而且通常可以重写你写出的任何内容。其结果是压制了极端。编辑产生的是第 95 百分位的写作——95% 的文章会因此改善，但 5% 的文章会被拖累。5% 的情况下你会得到“成群结队的极客”。

在网络上，人们可以发布任何他们想发布的内容。这些内容几乎都达不到经过编辑抑制的印刷出版物的水平。但作者的池子非常、非常大。如果池子足够大，缺乏抑制就意味着网上最好的写作应该超越印刷品中最好的。[[3]] 现在网络已经进化出了筛选好内容的机制，网络在净效果上获胜了。筛选胜过抑制，原因和市场经济胜过中央计划经济是一样的。

就连初创公司这次也和以前不一样了。它们和泡沫时期的初创公司之间的关系，就像博客和印刷媒体之间的关系。在泡沫时期，初创公司意味着一家由 MBA 领导的公司，烧掉几百万美元风险投资，以最字面意义上的方式“快速做大”。现在它意味着一个更小、、更技术化的团队，他们只是决定要做一个很棒的东西。他们以后再决定是否要拿风险投资规模的钱，如果拿的话，也会按照来拿。招聘已过时风险投资的挤压

3. 不要虐待用户

我想大家都会同意民主和 Ajax 是“Web 2.0”的组成部分。我还看到了第三个：不要虐待用户。在泡沫时期，很多热门网站对待用户都相当专横。不只是明显的方面，比如强迫他们注册，或者给他们看烦人的广告。90 年代后期普通网站的整体设计本身就是一种虐待。许多最热门的网站都充斥着突兀的品牌标识，导致加载缓慢，并向用户传达这样的信息：这是我们的网站，不是你的。（这在物理世界有一个类似物，就是某些笔记本电脑上贴的 Intel 和 Microsoft 的。）专为 Microsoft Windows 设计

我认为问题的根源在于，网站觉得自己是在免费赠送东西，而直到不久前，一家公司如果免费赠送任何东西，都可以相当专横。有时这达到了经济虐待狂的程度：网站所有者认为他们给用户造成的痛苦越多，自己获得的利益就一定越多。这种模式最戏剧性的残留可能是在 salon.com 上，你可以读到故事的开头，但要读完剩下的部分就必须坐着看完一段广告片。

在 Y Combinator，我们建议所有我们资助的初创公司永远不要对用户颐指气使。除非你需要存储某些东西，否则永远不要让用户注册。如果确实要让用户注册，也永远不要让他们等待邮件中的确认链接；事实上，除非有充分理由，否则连他们的邮箱地址都不要问。不要问他们任何不必要的问题。除非他们明确要求，否则永远不要给他们发邮件。永远不要把你链接的页面框在框架里，也不要在新窗口中打开它们。如果你同时有免费版和付费版，不要把免费版限制得太严。如果发现自己在问“我们要不要允许用户做 x？”只要不确定，就回答“应该允许”。在慷慨这一侧犯错。

在一文中，我建议初创公司永远不要让别人从自己下面飞过去，意思是永远不要让其他公司提供更便宜、更简单的解决方案。另一种低飞的方式是给用户更多权力。让用户做他们想做的事。如果你不这么做而竞争对手做了，你就有麻烦了。如何创办一家创业公司

iTunes 在这个意义上带有 Web 2.0 的色彩。你终于可以单曲购买，而不是必须买整张专辑。唱片 industry 讨厌这个想法，并尽可能长时间地抵制它。但用户想要什么很明显，所以苹果从唱片公司下面飞了过去。[[4]] 不过也许把它描述为 Web 1.5 更合适。如果把 Web 2.0 应用于音乐，大概意味着独立乐队免费赠送无 DRM 的歌曲。

对用户最好的方式，就是把竞争对手收费的东西免费给他们。在 90 年代，很多人可能认为我们现在应该已经有了一个可用的微支付系统。实际上事情朝着相反的方向发展了。最成功的网站是那些想出新办法免费赠送东西的网站。Craigslist 基本上摧毁了 90 年代的分类广告网站，而 OkCupid 看起来很可能对上一代约会网站做同样的事。

服务网页的成本非常非常低。如果你每页面浏览能赚到哪怕几分之一美分，你就能盈利。而且定向广告的技术在持续改进。如果十年后 eBay 被一个广告支持的免费 Bay（或者更可能叫 gBay）取代，我不会感到惊讶。

虽然听起来可能奇怪，但我们告诉初创公司，他们应该尽量少赚点钱。如果你能找到办法把一个十亿美元的产业变成五千万美元的产业，同时这五千万全归你，那就更好了。虽然确实，把东西变便宜往往最终会产生更多收入，就像自动化往往最终会创造更多就业一样。

最终的目标是微软。当有人通过提供免费的基于网络的 MS Office 替代品来戳破这个气球时，它会发出多大的声响啊。[[5]] 谁会来做呢？谷歌？他们似乎在慢慢来。我怀疑最终戳这个针的会是几个 20 岁的黑客，他们太天真以至于不会被这个想法吓倒。（这能有多难？）

共同主线

Ajax、民主、不轻视用户。它们有什么共同点？我直到最近才意识到它们有共同之处，这也是我如此讨厌“Web 2.0”这个词的原因之一。它似乎被用作一个标签，不管什么新的东西都往上贴——它并没有预测任何东西。

但确实存在一条共同主线。Web 2.0 意味着以网络本应被使用的方式来使用它。我们现在看到的这些“趋势”，不过是网络的本质从泡沫时期强加给它的那些错误模型下浮现出来而已。

我是在读到对 Joe Kraus——Excite 的联合创始人——的一次采访时意识到这一点的。[[6]]

Excite 其实从来就没有把商业模式搞对。我们掉进了经典的陷阱：当一种新媒介出现时，它会采用旧媒介的做法、内容和商业模式——而这是行不通的，然后人们才会去摸索更合适的模式。

在泡沫破裂后的那些年里，似乎没发生什么大事。但事后看来，有些事情确实在发生：网络正在找到它自然的休止角。比如民主这个组成部分——那不是一项创新，不是某个人刻意做出来的。那是网络自然会产生的东西。

把类似桌面应用的东西通过网络交付的想法也是如此。这个想法几乎和网络本身一样古老。但第一次尝试被 Sun 劫持了，我们得到了 Java applet。Java 后来被重塑为 C++ 的通用替代品，但在 1996 年，关于 Java 的故事是它代表了一种新的软件模型。你不再运行桌面应用，而是运行从服务器传来的 Java“applet”。

这个计划在自己的重量下崩溃了。微软帮助杀死了它，但它无论如何都会死掉。因为黑客们根本不买账。当你发现在推广某种东西，说它是下一个开发平台时，你可以肯定它不是。如果它是，你不需要公关公司来告诉你，因为黑客们早就已经在它上面写东西了，就像 Busmonster 这样的网站早在谷歌自己都没打算把它当作平台之前，就已经在拿 Google Maps 当平台用了。潜艇

Ajax 是下一个热门平台的证据是，成千上万的黑客自发地开始在它上面构建东西。Mikey 喜欢它。

像 del.icio.us 和 flickr 这样的网站允许用户用描述性的标签来“标记”内容。但它们忽略了一个巨大的隐性标签来源：网页链接中的文本。此外，这些链接构成了一张连接页面创建者（个人和组织）的社交网络，通过图论我们可以从这个网络计算出每个成员的声誉估计值。我们计划从网络中挖掘这些隐性标签，并将它们与它们所体现的声誉层级一起用来改进网络搜索。

你觉得他们平均要花多长时间才能意识到这其实是在描述 Google？

Google 是 Web 2.0 所有三个组成部分的先驱：用 Web 2.0 的术语来描述，他们的核心业务听起来非常时髦，“不要虐待用户”是“不要作恶”的子集，当然 Google 还用 Google Maps 开启了整个 Ajax 热潮。

Web 2.0 意味着以网络本应被使用的方式来使用它，而 Google 做到了。这就是他们的秘密。他们是顺风航行，而不是像印刷媒体那样坐在那里风平浪静地祈祷有个商业模式，或者像微软和唱片公司那样试图逆风而行、起诉自己的客户。[[7]]

Google 不会试图强迫事情按他们的方式发生。他们试图搞清楚什么将会发生，然后安排自己站在那里等着它发生。这才是对待技术的正确方式——而随着商业包含越来越多的技术成分，这也是做生意的正确方式。

Google 是一家“Web 2.0”公司这一事实表明，虽然这个词有意义，但它也相当虚假。它就像“对抗疗法”这个词一样。它只是意味着把事情做对，而当你需要一个专门的词来表达这个意思时，这本身就是一个坏兆头。
