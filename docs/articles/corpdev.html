别和企业发展部门谈

January 2015 · 阅读原文

2015年1月

企业发展部门（Corporate Development），简称 corp dev，是公司内部负责收购其他公司的团队。如果你正在和 corp dev 的人交谈，那基本上就是他们找你的原因——不管你现在是否已经意识到。

除非（a）你现在就想卖掉公司，而且（b）你很有把握能拿到一个可以接受的价格，否则和 corp dev 交谈通常是个错误。实践中，这意味着创业公司只有在发展得特别好或特别糟糕时才应该和他们接触。如果情况特别糟糕，公司即将倒闭，那不妨和他们谈谈，反正你也没什么可损失的。而如果你发展得特别好，则可以放心地和他们谈，因为双方都清楚收购价必须很高，如果他们表现出丝毫浪费你时间的迹象，你也会有足够的底气让他们滚蛋。

危险的是处于中间状态的公司。尤其是那些成立不久、增长很快、但还没壮大到一定规模的年轻公司。对于一家前景不错、成立不到一年的公司来说，哪怕只是和 corp dev 谈谈，通常都是个错误。

但创始人却经常犯这个错误。当 corp dev 的人想见面时，创始人会告诉自己至少应该去听听他们想干什么。而且，他们也不想因为拒绝见面而得罪大公司。

好吧，我来告诉你他们想干什么。他们想谈收购你。这就是“corp dev”这个职位的真正含义。所以在同意和 corp dev 的人见面之前，先问问自己：“我们现在想卖掉公司吗？”如果答案是否定的，就告诉他们：“抱歉，我们现在专注于公司的增长。”他们不会因此被冒犯。大公司的创始人也肯定不会生气。如果说有什么影响的话，他们反而会更看得起你。你会让他们想起当年的自己。他们当年也没卖公司；正因为如此，他们现在才有能力去收购其他公司。[[1]]

大多数被 corp dev 联系上的创始人其实都知道这意味着什么。然而即使他们清楚 corp dev 的工作内容，也明确知道自己不想卖公司，他们还是会去赴约。为什么？因为那种混合着否认和一厢情愿的心理，正是创始人大多数错误背后的根源。有人想收购你，和这样的人交谈本身就让人感到受宠若惊。而且谁知道呢，也许他们的报价会高得惊人。至少应该去看看报价，对吧？

不对。如果他们打算立刻通过邮件发给你报价，那当然可以打开看看。但和 corp dev 的对话不是这样进行的。如果你真的能拿到报价，也是在经历一段漫长且极其让人分心耗神的流程之后。而如果报价让人惊讶，那通常是低得惊人。

分心是创业公司最负担不起的事情。而和 corp dev 的谈话是最糟糕的一种分心，因为它不但会消耗你你脑海中的头号想法，还会摧毁你的士气。熬过一个艰苦过程的诀窍之一，就是不要停下来想自己有多累，而是让自己进入一种心流状态。[[2]] 想象一下，你在马拉松跑到第20英里时，有人突然跑到你旁边说：“你一定特别累吧。要不要停下来休息一下？”和 corp dev 的谈话就像这样，但更糟糕，因为“停下来”的暗示会在你脑海中和你想象中他们可能给出的高报价结合在一起。

然后你就真的麻烦了。如果有可能，corp dev 的人喜欢反客为主。他们喜欢把你逼到需要你去说服他们买，而不是他们说服你卖的地步。而且令人惊讶的是，他们经常能成功。

这是一条极其湿滑的下坡路，上面涂满了能影响创始人思维的最强大力量之一，而旁边还有一位经验丰富的专业人士，他的全职工作就是把你往下推。

他们把你往下推时使用的战术通常相当残酷。Corp dev 的人的全部工作就是收购公司，而他们甚至不能选择收购哪一家。衡量他们业绩的唯一标准就是能多便宜地把你买下来，那些更有野心的人会不择手段地达到这个目标。比如，他们几乎总是先扔出一个低得离谱的报价，只是为了看看你会不会接受。即使你不接受，一个过低的初始报价也会让你士气低落，更容易被操控。

而这还是他们最无辜的战术之一。等你已经谈好价格，自以为成交了，他们却回过头来说他们的老板否决了这笔交易，最多只愿意出之前谈定价格的一半。这种事屡见不鲜。如果你觉得投资人已经够恶劣了，那和 corp dev 的人比起来简直是小巫见大巫。即使是在其他方面还算善良的公司里，corp dev 的人也可能如此。

我记得有一次向我在 Google 的一个朋友抱怨，他们的 corp dev 团队对一家 YC 创业公司使了个很阴的招。

“你们不是说‘不作恶’吗？”我问。

“我觉得 corp dev 没收到那份备忘录，”他回答。

你在并购对话中遇到的战术，可能和你之前在硅谷这个相对刻薄的人会失败的世界里所经历过的任何事情都完全不同。就好像旧时代强盗资本家商业世界的一块基因材料被植入到了创业世界里。[[3]]

最简单的自我保护方法，就是用约翰·D·洛克菲勒（John D. Rockefeller，他的祖父是个酒鬼）用来防止自己酗酒的那个技巧。他曾经在一个主日学校课堂上说：

孩子们，你们知道我为什么从来没变成酒鬼吗？因为我从来没喝过第一口酒。

你现在想卖掉公司吗？不是将来，而是现在。如果不想，那就别去赴第一次约。他们不会因此被冒犯。而你则可以确保自己免于创业公司可能遭遇的最糟糕经历之一。

如果你确实想卖，那还有另一套方法。但创始人跟 corp dev 打交道时犯的最大错误，不是在准备好卖的时候没谈好，而是还没准备好就去和他们谈。所以如果你只记得这篇文章的标题，你就已经知道了关于创业第一年里并购（M&A）所需知道的大部分内容。

注释

[1] 我不是说你永远不应该卖公司。我是说你应该在自己头脑中明确是否真的想卖，不要因为被操控或一厢情愿而在原本不该卖的时候就试图卖掉。
[2] 在创业公司里，就像在大多数竞技体育中一样，手头的工作几乎会自动帮你做到这一点；你太忙了，以至于感觉不到疲惫。但当你失去这种保护，比如比赛结束哨声响起时，疲劳就会像海浪一样向你袭来。和 corp dev 谈话，就相当于让你在比赛进行中就感受到这种疲惫。
[3] 公平地说，corp dev 人员的明显不当行为有时会被放大，因为他们其实是大型组织的面孔，而这个组织往往自己都没想清楚要做什么。收购方对于收购可能出奇地优柔寡断，而当这种摇摆不定最终传递到你这里时，它和欺骗已经难以区分。感谢 Marc Andreessen、Jessica Livingston、Geoff Ralston 和 Qasar Younis 阅读本文的草稿。
