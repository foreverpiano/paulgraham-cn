能反击的过滤器

August 2003 · 阅读原文

2003年8月

我们或许可以通过让贝叶斯垃圾邮件过滤器跟随链接、查看另一端的内容来提升其准确性。death2spam 的 Richard Jowsey 目前已在临界情况下这样做，并报告说效果很好。

为什么只在临界情况下做？而且为什么只做一次？

正如我在 过滤器会消灭垃圾邮件吗？ 中提到的，如果跟踪垃圾邮件中的所有网址，会产生一个有趣的副作用。如果流行的邮件客户端为了过滤垃圾邮件而这样做，垃圾发送者的服务器将遭受沉重打击。我越想越觉得这是个好主意。这不仅仅有趣；很难想象还有比这更精准的反击垃圾发送者的方式。

因此，我想向那些正在开发垃圾邮件过滤器的人建议增加一项功能：一个“惩罚”模式，如果开启，它会对疑似垃圾邮件中的每个网址爬取 n 次，其中 n 可以由用户设置。[1]

正如许多人已经指出的，当前电子邮件系统的一个问题在于它过于被动。它只会按照你告诉它的去做。目前所有解决这个问题的建议似乎都涉及新的协议。而这个方案不需要。

如果得到广泛使用，自动抓取的垃圾邮件过滤器将让电子邮件系统实现反弹。垃圾邮件的巨大数量迄今为止一直对垃圾发送者有利，现在却会反过来对他不利，就像树枝猛地弹回打在他脸上一样。自动抓取的垃圾邮件过滤器会大幅提高垃圾发送者的成本，并降低其销量：他的带宽使用量会直线上升，服务器会在重压下几近停机，这将使那些原本可能回应垃圾邮件的人无法访问它们。

每小时发出一百万封邮件，你的服务器就会每小时承受一百万次点击。

我们需要确保这种行为只针对疑似垃圾邮件。通常情况下，任何被发送给数百万人的网址都很有可能是垃圾网址，因此对每封邮件中的每个 http 请求都进行提交，几乎所有时候都能正常工作。但也有少数情况并非如此：例如来自 Yahoo Mail 和 Hotmail 等免费邮件服务的邮件底部的网址。

为了保护这类网站，并防止滥用，自动抓取功能应当与被垃圾广告宣传的网站的黑名单结合使用。只有黑名单上的网站才会被爬取，而网站只有在经过人工检查后才会被加入黑名单。一封垃圾邮件的生命周期至少有几个小时，因此应该来得及更新这样的列表，以干扰推广新网站的垃圾邮件。[2]

高频次的自动抓取只对拥有高带宽连接的用户实际可行，但这样的人已经足够多，足以给垃圾发送者带来严重麻烦。事实上，这个解决方案巧妙地镜像了问题本身。垃圾邮件的问题在于，为了触达少数容易上当的人，垃圾发送者会把邮件发给所有人。不容易上当的收件人只是附带损害。但不容易上当的大多数人只有在能够阻止（或威胁要阻止）那些容易上当的人回应垃圾邮件时，才会停止收到垃圾邮件。自动抓取的垃圾邮件过滤器为他们提供了一种实现途径。

这是否能彻底消灭垃圾邮件？并不完全。最庞大的垃圾发送者或许能够保护他们的服务器免受自动抓取过滤器的影响。然而，对他们来说最简单、最廉价的应对方式，是在邮件中包含有效的取消订阅链接。而对于规模较小的垃圾发送者，以及那些雇佣垃圾发送者为自己推广的“合法”网站，这将成为一种必需。因此，如果自动抓取过滤器变得普遍，它们就会变成自动取消订阅过滤器。

在这种情况下，垃圾邮件就会像操作系统崩溃、病毒和弹出式广告一样，成为那些懒得使用正确软件的人才会遭受的祸害之一。

注释

[1] 自动抓取过滤器必须能够跟随重定向，并且在某些情况下（例如页面上只写着“点击这里”）应该跟随不止一层的链接。同时要确保 http 请求与流行网页浏览器的请求无法区分，包括顺序和 referrer。如果在 x 时间内没有收到响应，则默认给予相当高的垃圾邮件概率。与其让 n 保持固定，或许更好的做法是让它成为已见过提及该网站的垃圾邮件数量的函数。这将为防止滥用和意外提供进一步的保护。
[2] 本文最初版本使用的是“whitelist”一词而非“blacklist”。虽然它们的作用类似于黑名单，但我原本倾向于称其为白名单，因为这样可能使其不易受到法律攻击。不过这似乎只是让读者感到困惑。可能应该存在多个黑名单。单一故障点既容易遭受攻击，也容易被滥用。感谢 Brian Burton、Bill Yerazunis、Dan Giffin、Eric Raymond 和 Richard Jowsey 阅读本文的草稿。FFB FAQ 日文翻译 Perl 版 FFB Lycos DDoS@Home
