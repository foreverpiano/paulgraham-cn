提交累加器生成器

阅读原文

抱歉，暂时请不要再发送任何累加器生成器了。我没有时间去检查它们。

如果你想提交一个用新语言编写的累加器生成器示例，请先阅读这个页面。

我收到的大部分代码都无法运行，要么是因为作者误读了规范，要么是因为在对应的语言中实际上无法实现。

在提交示例之前，请确保 foo

接收且返回的函数都严格只接受一个参数。适用于任何数值类型——即可以接收整数和浮点数，并返回能同时接收整数和浮点数的函数。（仅仅将所有输入转换为浮点数是不够的。一个只接收过整数的累加器必须返回整数。）生成出的函数返回的是迄今为止所有传入数字的总和，而非仅最近一次的值。返回的是一个真正的函数，意味着你可以在程序中任意可以使用普通方式定义的函数的地方使用它。不会以可能导致累加值或返回的函数被其他代码意外修改的方式来存储它们。

例如，在示例之后，如果你添加以下代码（用一种虚构的语言）：

x = foo(1); x(5); foo(3); print x(2.3)

它应该输出 8.3。

我目前认为你无法用 ML 和 Ocaml 编写这个程序（缺乏多态性），Tcl（foo 返回的函数必须有名称，且没有可靠的方法避免名称冲突），或者 K（不支持闭包）。

我目前还无法确定是否可以用 Forth 编写这个程序。我已经收到了几个 Forth 提交，目前仍在努力分析它们是否正确。
