Python 悖论

August 2004 · 阅读原文

2004 年 8 月

在最近的一次 伟大的黑客 中，我说了一句话，惹恼了很多人：你能找到比 Java 项目更聪明的程序员来参与 Python 项目。

我这么说并不是指 Java 程序员笨。我的意思是 Python 程序员很聪明。学习一门新的编程语言需要投入大量精力。而人们学习 Python 并不是因为它能帮他们找到工作；他们学习它是因为真正喜欢编程，对自己已经掌握的语言感到不满足。

这恰恰使他们成为公司最应该聘用的那种程序员。因此，我就用一个暂时的名字——Python 悖论：如果一家公司选择用一种相对冷门的语言来编写软件，他们就能招到更好的程序员，因为他们只会吸引那些足够在意、愿意去学习它的人。而对程序员来说，这个悖论甚至更明显：如果你想获得一份好工作，那么应该学习的语言，恰恰是那些人们并非单纯为了找工作而学习的语言。

至今只有少数公司足够聪明地意识到了这一点。但这里也存在一种筛选：它们恰恰是程序员最想去工作的那些公司。比如 Google。当他们发布 Java 编程职位时，也希望应聘者有 Python 经验。

我有一个朋友几乎掌握所有主流编程语言，他的大多数项目都使用 Python。他说主要原因是他喜欢源代码的样子。这听起来像是挑选语言的一个很肤浅的理由。但其实没那么轻率：编程时，你花在阅读代码上的时间远多于编写代码。你像雕塑家摆弄黏土一样来回推动大块的源代码。因此，一门让源代码变得丑陋的语言，会让严谨的程序员感到抓狂，就像满是硬块的黏土会让雕塑家恼火一样。

一提到丑陋的源代码，人们自然会想到 Perl。但 Perl 那种表面上的丑陋并不是我所说的那种。真正的丑陋不是难看的语法，而是不得不用错误的概念来构建程序。Perl 可能看起来像卡通人物在骂街，但确实存在 书呆子的复仇 它在概念层面超越 Python。

至少目前是这样。当然，这两种语言都是 百年后的语言 的移动目标。但它们和 Ruby（以及 Icon、Joy、J、Lisp、Smalltalk）一样，都有一个共同点：创造它们和使用它们的人，都是真正关心编程的人。而这些人往往也正是那些能把编程做好的人。
