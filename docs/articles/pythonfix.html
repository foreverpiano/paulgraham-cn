Python 修复了什么

阅读原文

不少 Python 用户发来了关于“Python 修复了什么问题”的建议。

我仍然没有找到确切的答案。我知道现在人们选择 Python 的原因（因为 Perl 实在太乱），但我不确定这是否就是 Python 当初诞生的初衷。

George Demmy 的看法是：BASIC 太简陋（太弱鸡），Perl 太吓人（perlious? :)，而 Scheme 又太过诡异。

Python 之父 Guido van Rossum 曾多次公开表示——尽管这让许多 Python 忠实粉丝颇为不满——Python 的设计初衷是用来教从未接触过编程的人。我可以帮你找一些原话，但手头暂时没有现成的引用。

James Hague 认为：BASIC 和 Pascal 已经不再适合作为教学语言。

我知道这番话可能会让那些用 Python 开发大型项目的人感到不舒服——我也承认它确实是一门很适合干大事的语言——但如果你去追溯这门语言最初的创造动机，以及其诸多特性的背后理念，就会发现这种说法其实更有道理。

Laurent Szyster：C 太危险，Java 太死板，而 Perl 又太吓人。

Nathan Myers：Perl 就是一团糟。

Neil Schemenauer：ABC 不适合用作系统级的脚本语言。

Guido 在 CWI 工作期间参与了 ABC 项目。想了解更多细节，可以看看这篇访谈：http://www.amk.ca/python/writing/gvr-interview.html。
