SBL 也堕落了？

阅读原文

服务器黑名单往往会变质，因为它们赋予的权力会腐蚀管理它们的人。他们逐渐变成网络治安员，开始封禁的服务器。过滤器与黑名单

最臭名昭著的例子就是 MAPS RBL。我过去一直推荐 Spamhaus 的 SBL 作为替代方案，但现在看来，就连这个之前还算正直的服务，也正在步其他黑名单的后尘。

我最近收到的这封邮件，听起来异常耳熟：

你曾形容 Spamhaus SBL 是“负责任的”。恕我不敢苟同。我最近发现自己的 IP 段 ---- 被 Spamhaus SBL 列入了黑名单。我询问原因，他们告诉我，我的那个数据中心 ---- 过去曾经托管过一些垃圾邮件发送者。于是他们的整个网络（子网掩码 ----）都被拉黑了。我向他们解释，虽然我无法保证 ---- 的可接受使用政策（AUP）执行得有多严格，但在这件事上我是无辜的，并请求他们把我那四个 IP 从黑名单中移除。结果收到了一条相当粗鲁的回复，大意是“我们不会移除，因为是你自己选择了容忍垃圾邮件的 ISP”。我并不觉得这个回答有什么负责任的地方。

这对 SBL 乃至整个黑名单理念来说都不是好消息。SBL 创立的初衷，正是为了避免其他黑名单曾出现过的那些滥用行为。因此，如果连他们都走上了 MAPS RBL 的老路，我们恐怕只能推断：所有黑名单最终都会如此。
