什么是 Lisp？

Lisp 是一族编程语言，起源于 John McCarthy 在 1950 年代后期发明（或者更准确地说是发现）的一门语言。目前最主要的两个方言是 Common Lisp 和 Scheme。我们正在开发一个新的方言，叫做 Arc。Lisp 的根源

如何学习 Lisp？

学习任何编程语言的最好方法就是用它来写程序。你有两个主要方言可以选择：Scheme 和 Common Lisp。它们各有优缺点，但两者之间的差异远小于它们与其他语言之间的差异。所以如果你想开始学习 Lisp，选择哪个都无所谓。

两种方言都有很好的书籍，也有许多优秀的免费实现。因此，找一本好书和一个实现，然后开始尝试编写程序。阅读现有的源代码来了解 Lisp 的编程风格。如果遇到困难，可以在 comp.lang.lisp 论坛提问。那里的很多人总是乐于帮助新手。

我应该学 Common Lisp 还是 Scheme？它们有什么区别？

Common Lisp 功能强大但外观不够优美。Scheme 小巧而简洁，但其标准只定义了语言的核心。如果要交付实际应用，我可能会选择 Common Lisp；如果要教课，我可能会用 Scheme（但会用到 Common Lisp 的宏）。

你对 ML 及其衍生语言怎么看？

我认识的大多数黑客都对 ML 家族感到失望。如果程序是你在事先完全想好，然后只是翻译成代码，那么静态类型语言会更合适。但实际写程序并不是这样的。

无法拥有混合类型的列表是一个特别严重的限制。它妨碍了探索式编程（早期把所有东西都表示为列表会很方便），而且意味着你无法拥有真正的宏。
