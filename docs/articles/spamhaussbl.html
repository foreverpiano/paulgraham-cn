又一个 SBL 的故事

阅读原文

（我收到了一家小型 ISP 的邮件。发件人要求我隐去他的姓名，以免遭受报复。）

我管理一家小型 ISP，多年以来一直使用 Spamhaus 的 SBL 列表。我们自 1995 年起就从事 ISP 业务，正因如此，我们得以在 IP 地址还能购买的时候买下了自己的 IP 地址块，而不是像现在这样只能租用。

我们最近把网络中的部分 IP 租给了一些选择加入的营销商。后来我们发现，这些家伙竟然是全球最大的垃圾邮件发送者之一。（他们通过第三方出面，隐瞒了自己的真实身份，直到开始大规模群发邮件才暴露。）

他们使用的 IP 地址已经从我们的网络中移除，通过 SWIP 转给了他们，并通过知名全球网络提供商进行路由。当我们开始收到 UCE 投诉时，便立即着手将这些 IP 收回到自己的网络。其中一位客户由于租赁协议写得不够严谨，加上我们之前的 AUP 政策较为宽松，成功拖延了我们好几个月。我们可以在自己的网络内阻止垃圾邮件发送者，但当 IP 被托管在他人网络上时就无能为力了。最终我们还是成功让承租人离开了 IP 段并收回了地址，但造成的损害已经无法挽回。

现在我们被 Spamhaus 标记为“垃圾邮件支持服务商”，尽管按照他们自己的定义，这一标签根本不适用。我们甚至不想提 Spamhaus 在 SBL 条目中对我们公司发布的那些不准确、近乎诽谤的内容。我们被列入黑名单，据说是为期六个月，每天都在不断失去客户。

Spamhaus 垃圾邮件支持服务定义：

为垃圾邮件活动提供“防弹”托管的服务、运营“垃圾邮件软件”网站，或明知是为垃圾邮件活动提供服务的行为。

我们并没有提供任何托管服务，也没有运营任何网站（无论是垃圾邮件软件还是其他），而且我们也并非“明知”是在为垃圾邮件活动提供服务。

事实上，当我们发现这些人在从事垃圾邮件活动后，立即向 Spamhaus 通报了相关 IP 段，并以最快速度收回了地址（在不涉及法律问题的情况下）。而 Spamhaus 的回应却是升级了黑名单条目，将我们的整个 IP 地址块全部列入黑名单——总共 1.6 万个 IP，而不仅仅是那几百个发出垃圾邮件的 IP。他们的回复是：“当一家 ISP 联系我们报告垃圾邮件源时，我们就会直接把这家 ISP 拥有或租用的所有 IP 都加入黑名单。”

根据他们的除名政策：一旦收到 IP 拥有者（互联网服务提供商）发来的通知，表明垃圾邮件活动已经终止，相关 IP 将立即从 SBL 中移除。

真是讽刺！

我们的 ISP 今年迎来十周年，正在本地市场提供服务。我们从未允许垃圾邮件从我们的网络发出，也永远不会允许。我们真诚地采用了 Spamhaus 的 AUP，因为它涵盖了 ISP 可能遇到的各种问题。我们向 Spamhaus 说明了这一点以及其他情况，而他们给我们的回复基本上是：我们认为 IP 所有者如果价钱合适还会再次出租，但如果你们能保持干净六个月，我们就会解除对你们 IP 的封锁。

事情还没完。在我们从 SWIP 位置收回所有 IP 之后，我们采取措施从上游租用 IP，这样即使 Spamhaus 继续封锁我们的 IP，我们合法用户的邮件也能正常投递而不受影响。这一做法效果很好，大约持续了三周。直到我们发现，Spamhaus 又把我们上游提供商的 IP 也列入黑名单，理由居然和我们被封的 IP 有关。再强调一次，我们的服务器从未发送过垃圾邮件，除非是偶尔有用户电脑感染了病毒或木马。那么 Spamhaus 为什么要封这些 IP 呢？我们不得而知。

于是我们又安排将邮件发送到一台中继服务器上，这是一台封闭的中继服务器，只允许我们的邮件服务器通过它中转。现在，连这台允许我们中继的邮件服务器也被 Spamhaus 封锁了。

就这样，我们持续失去那些已经跟随我们十年的老客户，只因为一个人凭自己的主观判断，认为我们是一家垃圾邮件支持服务商。
