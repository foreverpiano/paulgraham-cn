过滤器会消灭垃圾邮件吗？

December 2002 · 阅读原文

2002年12月

（本文改编自我在《计算机安全期刊》2003年1月刊上发表的一篇文章。）

我每天大约收到45封垃圾邮件，但只有大约每周一封会进入我的收件箱。如果每个人都能把这么多垃圾邮件过滤掉，垃圾邮件发送者就会放弃继续发送了。

这会发生吗？

第一代垃圾邮件过滤器使用规则来识别特定的垃圾邮件特征。现在，新一代的统计垃圾邮件过滤器似乎提供了显著更好的性能。统计过滤器会查看每封 incoming 邮件的全部内容，并根据它与以往垃圾邮件的整体相似度来判断是否为垃圾邮件。这种新型过滤器通常能捕获当前99%以上的垃圾邮件，且几乎不会产生误报。

最简单的统计过滤器可以用一段话来描述。用户将所有垃圾邮件丢到单独的垃圾箱中。每隔一段时间，一个程序会扫描用户的所有邮件，对于每个 token，计算其在垃圾邮件中出现的次数与总出现次数的比率。例如，如果“cash”在1000封垃圾邮件中出现200次，在500封非垃圾邮件中出现3次，那么它的垃圾邮件概率是

(200/1000) / (3/500 + 200/1000)

即0.971。当一封新邮件到来时，提取所有 token，找出概率 p1...p15 中距离0.5最远（无论哪个方向）的15个。邮件是垃圾邮件的概率为

p1p2...p15 ------------------------------------------ p1p2...p15 + (1 - p1)(1 - p2)...(1 - p15)

我使用0.9作为阈值，但阈值设在哪里其实没那么重要，因为大多数概率最终都会接近0或1。

在过去几个月里，许多新的统计垃圾邮件过滤器纷纷出现。现在已有超过30种可用。Apple 有一个，MSN 有一个，据说 AOL 有一个处于测试阶段，你可以相当肯定 Yahoo 也在开发一个。

其中一些过滤器受到我写的一篇名为 反垃圾邮件计划 的文章启发，但两个开源的统计文本分类器（spam 过滤的超集）——ifile 和 CRM114——已经存在多年，而 Apple 和 Microsoft 也已经在统计过滤上工作了一段时间。统计过滤器突然爆发式出现的原因，可能只是因为垃圾邮件已经严重到足以让人们开始认真关注它，而当你认真对待时，得到的自然就是统计过滤器。

这些过滤器的工作方式并不完全相同。上面描述的算法称为“朴素贝叶斯分类器”，因为它使用贝叶斯定理的一个退化形式来组合概率。大多数新的开源过滤器都是朴素贝叶斯的，我猜测 MSN 的也是。Apple 的过滤器基于“自适应潜在语义分析”来计算垃圾邮件概率，在我看来这本质上是一回事。

无论它们如何计算概率，这些新的统计过滤器都拥有一些重要的共同优势：

1. 它们非常有效。即使是最简单的统计过滤器也能捕获当前99%的垃圾邮件。我所知道的最有效的过滤器——Bill Yerazunis 的 CRM114——能捕获99.8%。（我的稍落后，大约99.7%。）

2. 它们产生的误报很少。误报，即合法邮件被错误地当作垃圾邮件，是垃圾邮件过滤的致命弱点。统计过滤器产生的误报更少，因为它们既考虑有罪证据，也考虑无罪证据。一个在你的非垃圾邮件中不成比例频繁出现的 token，比如朋友的名字，其降低垃圾邮件概率的作用，与“cash”这样的 token 增加概率的作用一样大。

3. 它们会学习。你不需要翻阅成堆的垃圾邮件并制定规则来识别它们。无论里面有什么，过滤器往往都能发现它。和我们一样，统计过滤器会注意到 token “cash”是垃圾邮件的信号。然而，它们还会注意到“modalities”（在尼日利亚垃圾邮件中出奇高比例地使用）和“FF0000”（代表亮红色的 HTML）是更好的垃圾邮件信号。而且随着垃圾邮件发送者改变他们的信息或基础设施，过滤器也会随之适应。

4. 它们让每个用户定义什么是垃圾邮件。虽然统计过滤器可以在网络层面使用，但理想情况下概率应该针对每个用户单独计算。在用户对垃圾邮件的定义存在差异的程度上，他们的收件箱也会反映这一点。

5. 它们难以被欺骗。只有两种方法能绕过统计过滤器：使用更少的坏词，或者使用更多的无辜词。垃圾邮件发送者无法做到后者，因为最无辜的词（与你的朋友和家人、工作、兴趣相关的词）对每个用户来说都是不同的。因此他们只能使用更少的坏词。他们不能使用奇怪的拼写（例如用“Freee”代替“Free”），因为过滤器很快就会学会识别这些。他们唯一的选择是使用越来越模糊的委婉语，或者干脆使用一些听起来通用的文本，再加一个链接。

当这一代新的垃圾邮件过滤器交付给终端用户时，会发生什么？最令人兴奋的可能性是，它们可能会让垃圾邮件消失。

垃圾邮件发送者最关心的是响应率。在任何类型的直销中，收入都与响应率成正比。垃圾邮件发送者对响应率的要求比直邮低得多，因为他们的成本要低得多，但响应率仍然是他们赚钱多少的关键。

过滤直接击中了垃圾邮件发送者的要害：如果收件人看不到垃圾邮件，他们就不会回应。如果我们能过滤掉95%的垃圾邮件，就会使垃圾邮件发送者的收入减少到原来的1/20。如果我们能过滤掉99.5%，收入就会减少到原来的1/200。垃圾邮件发送者的成本很低，但也没有低到那个程度。在《底特律自由报》的一篇文章中，一位垃圾邮件发送者说他向2500万个地址发送邮件收取固定费用22,000美元。如果过滤器将响应率降低100倍，那么他所销售物品的平均价值将降至220美元。我怀疑这甚至无法覆盖他的成本。

过滤器至少应该让我们免于看到大部分垃圾邮件。但如果它们能足够大幅降低垃圾邮件发送者的响应率，垃圾邮件将不再有利可图，发送者实际上就会停止发送。

我们希望如此。但这里存在一个令人担忧的可能性。如果邮件程序设计不当，垃圾邮件仍然可能被垃圾邮件发送者最想触及的那些人看到。

回应垃圾邮件的人是极少数。响应率可能低至百万分之十五。这就是整个问题所在：垃圾邮件发送者浪费百万人的时间，只是为了触达那15个最愚蠢或最变态的人。

如果我们想让垃圾邮件失效，就必须设法阻止那15个蠢货回应发送给他们的垃圾邮件。否则垃圾邮件发送者就会继续向所有人发送。所以，奇怪的是，过滤是否能消灭垃圾邮件，完全取决于那15个蠢货怎么做。

最大的危险是，在蠢货市场中最广泛部署的任何过滤器，都会要求用户付出太多努力。贝叶斯过滤器根据每个用户收到的垃圾邮件和非垃圾邮件来计算垃圾邮件概率。这需要有两种删除方式：普通删除和“作为垃圾邮件删除”，用户必须以正确的方式删除每种消息。问题在于，那15个蠢货可能也正是那15个懒得这么做的用户。

只要这15个蠢货继续看到垃圾邮件，我们所有人就都会继续收到它们。因此，过滤器能否终结垃圾邮件，完全取决于蠢货们使用的邮件软件是如何设计的。我猜蠢货们都相当被动，所以关键在于让默认设置做正确的事。听我说，AOL 和 Microsoft：当你们发布贝叶斯过滤器时，不要让所有用户都从零开始训练自己的过滤器。使用基于你们所有用户分类邮件的初始过滤器。这样，只要用户一直盲目点击，大部分邮件最终就会进入正确的语料库。

这样做，垃圾邮件就会减少，这意味着基础设施成本降低，从而给你们带来更高的利润。

不过，还有另一种可能性我们应该担心，尽管可能没那么严重。如果那15个蠢货在垃圾邮件文件夹里看到垃圾邮件并做出回应呢？我认为这种情况不会经常发生。我们可以从垃圾邮件发送者竭力防止自己的邮件进入 Yahoo Mail 和 Hotmail 等服务的垃圾邮件文件夹的努力中看出这一点。但邮件软件的设计者至少应该记住这一点。悄悄清空垃圾邮件文件夹，不要鼓励用户去翻看并打开那些邮件。（一个好办法是在每封垃圾邮件的主题行旁边，随机顺序列出一些罪证性词汇。这样用户就不会被诱惑去打开主题模糊的垃圾邮件。）

如果邮件软件正确实现了它们，统计过滤器应该会将几乎所有垃圾邮件直接送入垃圾文件夹，这将大幅降低响应率。

受过滤器威胁最大的是 opt-in 垃圾邮件发送者。Opt-in 垃圾邮件发送者处于垃圾邮件光谱中更合法的一端。他们从其他公司购买你的地址，那些公司的服务条款（你在第47段没注意到吗？）允许他们出售客户的电子邮件地址。他们被称为 opt-in 垃圾邮件发送者，是因为他们通常在发给你的邮件中声称是你要求接收他们的邮件。

最大的 opt-in 垃圾邮件发送者之一声称拥有6000万个独特的电子邮件地址，几乎全部是国内的。6000万个地址超过美国在线人口的一半。如果一半的美国互联网用户都要求接收这家公司的邮件，那么应该很容易找到一个这样做过的人。然而，如果你开始在朋友中询问，我敢打赌你一个也找不到记得自己要求接收他们宝贵优惠的人。

更好过滤器的到来将终结 opt-in 的谎言，因为 opt-in 垃圾邮件特别容易被过滤器捕获。Opt-in 垃圾邮件发送者不会试图隐藏自己的身份。他们不像真正的 hardcore 垃圾邮件发送者那样使用假的发件人地址。邮件文本会公开说明它来自谁。因此 opt-in 垃圾邮件非常容易过滤。最简单的统计过滤器就能100%捕获它。

一旦统计过滤器被广泛部署，大多数 opt-in 垃圾邮件就会直接进入垃圾箱。这应该会把 opt-in 垃圾邮件发送者从他们目前半合法的掩护中赶出来。如果过滤破坏了他们的响应率，他们要么会倒闭，要么不得不承认自己的真实身份，并开始采取措施隐藏自己消息的来源和性质。

所有垃圾邮件发送者如果想绕过统计过滤器，都将不得不这么做。他们能做到吗？

我们已经看到大量证据表明，垃圾邮件发送者会调整他们的邮件，以绕过基于特定词或模式的简单垃圾邮件过滤器。他们能做什么来绕过统计过滤器，这是否会有效？

如果统计过滤器注意到你在使用垃圾词，你就有麻烦了。因此垃圾邮件发送者有两个选择：阻止过滤器看到这些词，或者使用不同的词。

垃圾邮件发送者至少在六个月前就一直在试图阻止过滤器识别他们邮件中的词。最常见的伎俩是使用变体拼写——例如用0代替O，用1代替l。一旦语料库达到一定规模，这些对贝叶斯过滤器来说就不是问题了，而且实际上有助于识别垃圾邮件。

垃圾邮件发送者还试图通过将 token 拆开来阻止过滤器识别邮件中的 token——例如，在单词中间使用空格或标点符号

li ke th.is

但这也不太有效。一个原因是合法邮件中不会有很多单独的字母或词片段，因此像“ke”或“th”这样的片段往往会有高于平均水平的垃圾邮件概率。另一个原因是他们无法对头信息和 URL 进行这种混淆，而这些本身就足以识别大多数垃圾邮件。如果必要，我们大概可以重建被拆开的单词，但到目前为止这甚至都没有必要。

垃圾邮件发送者有时会在单词内随机位置插入 HTML 注释，但这也很容易忽略。总的来说，在 token 方面，这是一个堵漏洞的问题。垃圾邮件发送者能用的伎俩数量有限，我们可以逐个应对。到目前为止还没有哪个是无法克服的。

人们有时会问，如果垃圾邮件发送者把邮件作为图片发送呢？他们已经在这样做了，而这种垃圾邮件很容易被过滤器捕获。像“img”和“href”这样的 token 具有类似于色情词汇的垃圾邮件概率。此外还有 URL 中的域名和文件名，以及一如既往的头信息。总体而言，包含 HTML 的垃圾邮件很容易过滤。最顽固的垃圾邮件发送者似乎知道这一点，并且已经在他们的邮件中避免使用 HTML。无论未来的垃圾邮件是什么样子，它很可能都不会包含 HTML。

我认为垃圾邮件发送者剩下的唯一领域将是词汇。他们无法阻止过滤器看到他们正在使用的词，所以他们将不得不使用不同的词。

简单地用随机词来填充垃圾邮件是行不通的。在我前面描述的算法中，只有统计意义上最重要的15个词会影响概率，而像“onion”这样的中性词，无论有多少，都无法与罪证性的“viagra”竞争统计显著性。为了压过罪证性词，垃圾邮件发送者需要用特别无辜的词来稀释他们的邮件，即那些不仅中性，而且在用户合法邮件中不成比例频繁出现的词。但这些词（朋友和家人的名字、工作中的术语）对每个收件人来说都是不同的，而垃圾邮件发送者没有办法弄清楚它们是什么。

我认为对垃圾邮件发送者唯一有效的做法是避免使用罪证性词汇。有趣的是，这对不同类型的垃圾邮件有效程度不同。对于抵押贷款垃圾邮件来说，这将是致命的。你怎么能不使用“mortgage”、“rate”、“lend”和“borrow”这样的术语来向人们推销抵押贷款呢？而其他类型的垃圾邮件则可以被改写成完全不使用可识别的垃圾词。

一种相当常见的垃圾邮件类型是由在线约会服务（抱歉，是在线约会服务的不负责任的联盟伙伴）发送的，假装是某个想认识你的女性会员发来的邮件。只要注册并搜索她的资料。这类邮件的文本中可能几乎没有什么能让过滤器注意到的内容。我见过一些，其文本在统计上与真实的人可能发给你的邮件无法区分。在这种情况下，你唯一能依靠的是 URL 和头信息。

当然，URL 和头信息已经提供了很多依据。我认为过滤最终会依赖于这些。我预测未来的垃圾邮件会是这样的：

To: joeuser@aol.com

From: john@smith.com

Subject: hey

Check out this site.

http://www.randomdomain.com

也就是说，一些相当 innocuous 的文本，后面跟一个链接。链接中的域名必须是以前没用过的，否则过滤器就会认出来。

这种类型的垃圾邮件响应率会是多少？这是最大的问题。它肯定会低于充满销售宣传的垃圾邮件，否则垃圾邮件发送者就不会费心写那些销售宣传了，但会低多少呢？如果响应率足够低，那么迫使垃圾邮件变得如此通用就足以消灭它。

在垃圾邮件能被做成如此通用且仍然有效的情况下，我们仍然有技术可以识别它。我能想到两种处理未知域的方法。一是派出一个爬虫去查看被引导到的页面。你可以像过滤邮件正文一样过滤页面。如果被广泛使用，这会产生一个有趣的效果：能反击的过滤器 任何在垃圾邮件中被大量广告的网站。如果过载的服务器需要一段时间才能响应，这本身就是证据——一种点对垃圾邮件发送者对点的过滤网络。

另一种处理未知域的方法是建立一个服务器，过滤器可以向其报告在垃圾邮件中看到的域名。对新 URL 不确定的过滤器可以查询这个服务器并快速得到答案。在链接指向正在传播的某个图片或新闻故事的情况下，你可能会得到误报，但那种邮件通常是由已经在你白名单上的朋友发送的。

而且别忘了，还有头信息。目前，垃圾邮件发送者在头信息上到处留下指纹。有些垃圾邮件软件实际上会在头信息中用名称标识自己。总有一天我们会为这种古朴感到好笑。但即使在他们清理掉明显的东西之后，垃圾邮件发送者还是会在一段时间内倾向于使用相同的基础设施。除非他们使用开放中继。那就太复古了。但我认为如果他们不得不求助于那种方式，会让很多垃圾邮件发送者破产。当然，像 Disney 这样的客户会对此望而却步。

奇怪的是，我预计未来会看到的垃圾邮件会很像过去的垃圾邮件。它会更少，就像过去那样。而且垃圾邮件发送者会是真正的亡命之徒，就像在过去几年垃圾邮件变得企业化之前那样。

我不想低估垃圾邮件发送者的 ingenuity。他们之前已经闯过了我们设置在他们面前的每一个障碍。但我认为这并非不可避免。我认为我们只是直到最近才真正下功夫设置障碍。统计垃圾邮件过滤器代表了一种更认真的反击努力。如果它们不能终结垃圾邮件，至少也能确保我们看到得更少。
