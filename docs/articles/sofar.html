到目前为止，一切顺利

August 2003 · 阅读原文

2003 年 8 月

（距离反垃圾邮件计划的发表已经快一年了。到目前为止，过滤器正占据上风。本文分析了垃圾邮件发送者为了绕过过滤器所尝试的各种伎俩，并对未来提出了一些建议。）

贝叶斯过滤器如今已经相当普及，我们开始看到专门设计来绕过它们的垃圾邮件。到目前为止，这些伎俩并未奏效。我的过滤成功率依然超过 99.7%，而 Brian Burton 报告他的多词贝叶斯过滤器 SpamProbe 达到了惊人的 99.9%。

这样的过滤率能否持续下去？垃圾邮件发送者正在采取什么手段攻击贝叶斯过滤器？我们能否做些什么来让过滤器变得更加严密？

更多优质 Token

要绕过贝叶斯过滤器只有两种方法：增加更多优质 token，或者减少不良 token。垃圾邮件发送者正在积极尝试这两种策略。

他们试图通过插入随机词典单词，或附加一大段中性文本（通常来自书籍或通讯社文章）来增加优质 token。但这两种伎俩效果都不佳。

随机选择的单词产生的效果（正如你可能预料到的）是，它们在垃圾邮件中出现的概率和在正常邮件中差不多。垃圾邮件的词汇量比正常邮件略窄，因此随机添加单词可能会给垃圾邮件发送者带来轻微的好处，但从统计角度看基本是相互抵消的。

大多数随机选择的单词在垃圾邮件和正常邮件中都没有出现过，因此它们的垃圾邮件概率是中性的。（我仍然将未见过单词的默认概率设为 0.4。）你可以通过设置出现次数阈值来抵消随机 token 带来的噪音，正如反垃圾邮件计划中所推荐的那样。我仍然使用 3 次的阈值。

附加文章或书籍片段的做法似乎也没什么效果，至少在我见过的案例中是这样。附加的文本看起来不像垃圾邮件，但也不太像我收到的正常邮件，因此从统计上讲往往没有太大影响。

许多垃圾邮件发送者现在会在“发件人”一行中使用随机生成的名字，但这反而让过滤变得更容易：我收到很多陌生人的邮件，但迄今为止还没有人叫 Krystal 或 Louella。我认为大多数用户的联系人的名字会落在某个小而稳定的子集中。因此随机选择名字会产生具有高垃圾邮件概率（而非中性）的 token。

减少不良 Token

欺骗贝叶斯过滤器的另一种方式是使用更少的不良 token。有两种主要策略：试图隐藏不良 token，或者改写邮件以使用不那么明显的垃圾邮件语言。

到目前为止，试图隐藏不良 token 的做法完全失败了。我见过的所有这类伎俩都让垃圾邮件更容易被抓住，而不是更难。这些伎俩包括故意拼错单词（V1agra）、用空格或 HTML 拆分单词（S E X）、以图片而非文本形式发送垃圾邮件，以及发送一个生成垃圾邮件的 Javascript 程序。

拼错的单词最终具有比它们想要隐藏的原词更高的垃圾邮件概率。在我的过滤器中，“Viagra”的垃圾邮件概率是 0.9848，而“V1agra”则是 0.9998。[1] 要让这种伎俩生效，你必须是第一个在垃圾邮件中使用几乎每一种错别字的人。这种概率很低，而且一旦失败，你只是教会了过滤器所有新的错别字。

拆分单词也有同样的效果。让分词器忽略大多数这类伎俩并不难，但即使你不这么做可能也没关系。正常邮件中不会包含被拆分的单词片段，因此它们会迅速获得很高的垃圾邮件概率。在我的过滤器中，主题行中单独出现的字母 S 的垃圾邮件概率为 0.9427。

以图片而非文本形式发送垃圾邮件同样无效，因为显示图片需要某些 HTML 标签，而这些标签最终都会具有极高的垃圾邮件概率。尤其是 URL。如果你使用一个曾经出现在垃圾邮件中的域名，那你就完蛋了。如果你改用 IP 地址，那就死得更彻底。没有哪种 token 的垃圾邮件概率能高于 URL 中的数字。

将垃圾邮件隐藏在 Javascript 程序中同样会因为类似的原因失败。Javascript 在正常邮件中比 img 标签还要罕见，因此 Javascript 程序中的 token 会获得极高的垃圾邮件概率。

改写

用不那么明显的垃圾邮件语言改写邮件是这些策略中唯一有可能成功的。但这需要大量工作。对某些垃圾邮件来说甚至可能无法实现。你如何在不使用“refinance”（0.9612）、“lenders”（0.9862）或“mortgage”（0.9995）这类术语的情况下改写一封贷款垃圾邮件？而且记住，无论你使用什么委婉语，它们都必须不同于之前所有贷款垃圾邮件发送者用过的。到了这个地步，垃圾邮件发送者改行做更合法的生意可能反而更省力。

这是一个重要的考量。如果绕过贝叶斯过滤器的唯一方法是更聪明地撰写垃圾邮件，那我们就让发送垃圾邮件变得困难得多，因为我们把“聪明”的负担从少数相对聪明的垃圾邮件软件编写者，转移到了大量更笨拙的实际撰写垃圾邮件的人身上。

垃圾邮件的基础设施是由聪明人构建的——也就是 Jon Praed 所说的“堕落的黑客”。然而垃圾邮件本身却是由单个垃圾邮件发送者撰写的。垃圾邮件软件对他们的帮助也有限。它可以为他们往邮件里插入随机词、拆分或拼错单词，但它无法用不那么明显的语言改写垃圾邮件。那需要人工智能才能做到。

当垃圾邮件发送者确实尝试改写他们的信息时，他们可能会通过用更中性词语组成的短语替换单个明显的垃圾 token 来实现。但多词过滤器也会学习并捕捉这些短语。Brian Burton 的 SpamProbe 和 Bill Yerazunis 的 CRM114 已经在考虑多词模式。当垃圾邮件通过我的过滤器时，我会把它发给它们，它们似乎总能抓住。

当然，如果垃圾邮件必须用更中性的语言改写，它们就不会那么有效。会回复垃圾邮件的人大概相当迟钝，必须用大量大写字母和感叹号猛击他们的脑袋才能让他们采取行动。除非你告诉他们必须 ACT NOW！否则你可能根本无法让他们行动。所以迫使垃圾邮件发送者使用更中性的语言可能就足以让其中大多数人关门大吉。我们将在未来一年内看到结果。

未来

这场战斗才刚刚开始。我看到明显针对贝叶斯过滤器的垃圾邮件也只有几个月时间。但既然 AOL 已经发布了贝叶斯过滤器，我们将会看到更多这类邮件。

这场战斗将如何展开？

正如我在《垃圾邮件计划》中所说，我认为最终可能都归结于链接。网络才是垃圾邮件的主要根源，而非电子邮件。几乎所有垃圾邮件都包含某种联系方式，而这几乎总是链接。[2] 这才是过滤器开发者应该重点关注的部分，因为这是垃圾邮件发送者最难改变的部分。

更好的贝叶斯过滤提到，通过给 token 标记上下文可以让过滤器更具区分度。主题行中的“FREE”比正文中的“FREE”具有高得多的垃圾邮件概率（0.9999 vs 0.7567）。目前我只给 token 标记了它们所在表头的名称。这个想法可以进一步扩展，从链接中榨取更多信息。链接内的“here”、“remove”或“img”等 token 将具有比平时高得多的垃圾邮件概率。（这对于例如 font 标签内的 token 可能也成立。）

为了应对过滤器，垃圾邮件的体积正变得越来越小。在许多邮件中，有效载荷现在只是链接内的一个单独图片标签：

&amp;lt;a href="http://sea2ws.com/gp/default.asp?id=011"&amp;gt; &amp;lt;img src="http://pharmhere1.com/pics/gp1.jpg"&amp;gt;&amp;lt;/a&amp;gt;

但即使是这么多 HTML 也足以抓住垃圾邮件。实际上，你可以根据其形式来识别这类垃圾邮件。

如果这些垃圾邮件中除了链接和图片标签外还有其他内容，通常只是填充物——用来欺骗过滤器的随机文本。到目前为止，这种填充物效果不佳，但我们不能假设它会一直如此。有办法生成能更好地抵消链接和图片垃圾邮件特征的文本。

最终过滤器可能需要采取措施来识别并忽略填充物。例如，如果一封邮件有一些垃圾邮件概率极高的 token，同时又有垃圾邮件概率极低的 token，你可能会想询问这些明显的垃圾词是否都聚集在一起，或者只查看 HTML 部分来重新评分。

另一种剔除填充物的方法是查看文本是否看起来符合语法。你不一定需要完整解析它。至少对于英语，我能想象出几种快速得出统计估计的方法。

最难捕捉的垃圾邮件是我所说的“未来垃圾邮件”——少量纯文本加一个 URL：

Hey there. Check out the following: http://www.blackboxhosting.com/foo

未来已经到来。我现在经常看到这样的垃圾邮件。我仍然能抓住几乎所有这类邮件——仅靠邮件头就足以抓住目前大部分垃圾邮件——但我漏掉的那 0.3% 的垃圾邮件主要就是这类“未来垃圾邮件”。

在未来垃圾邮件中，销售说辞被推后了一步。它不再像普通垃圾邮件那样包含在邮件本身中，而是等待一次点击，位于某个网站上。

这种趋势令人鼓舞，因为它意味着过滤器正在获胜。垃圾邮件实际上正在退却。（这不仅仅是象征性的胜利；每多一步都会降低回应率。）

如果垃圾邮件内容在网站上等待，为什么不让过滤器去查看那里有什么呢？你可以几乎不做改动地将过滤算法应用到网站内容上。death2spam 的 Richard Jowsey 已经在边缘案例中开始这么做，他报告说效果很好。

一种更廉价的替代方案是建立一个合作列表，收录在垃圾邮件中被宣传的网站。过滤器不必检查网站本身，而是可以查询一个服务器（或 P2P 网络），看看其他用户是否最近报告过推广该域名的垃圾邮件。如果有，就可以将其视为一个具有极高垃圾邮件概率的 token。（一个合作维护的垃圾广告网站列表对能反击的过滤器也很有用。）

这个想法可以推广到一个合作列表，收录所有具有高垃圾邮件概率的 token，而不仅仅是域名。这将提高所有人的过滤率，特别是对新安装过滤器的用户，他们现在几乎不需要什么训练。

为了利用这类信息，我们理想情况下应该尽可能延迟过滤。也就是说，在用户检查邮件时而不是邮件到达服务器时进行过滤。到你查看邮件时，任何进入你收件箱的垃圾邮件很可能已经被成千上万的人看到过了。

分而治之

有些人认为垃圾邮件发送者最终会找到绕过过滤器的方法，因此更好的解决方案是通过法律将发送垃圾邮件定为联邦犯罪。我自己也很乐意看到这样的法律——前提是它们写得恰当。而且奇怪的是，我认为过滤器将有助于实现这一点。有一类垃圾邮件发送者即使知道如何也无法躲避过滤器，消灭他们将更容易打击剩下的那些。

现在很难通过有效的反垃圾邮件法律，因为垃圾邮件发送者形成了一个连续谱，从（咳咳）“基于许可的邮件营销者”（如 Virtumundo，他们向从隐私政策不严谨的网站购买的地址发送未经请求的邮件），到最底层的如 Alan Ralsky，他们向从网页、聊天室和新闻组收集的地址发送未经请求的邮件。[3]

处于更合法一端的公司会游说设置漏洞，让底层发送者也能钻空子。例如，国会似乎在考虑允许未经请求的邮件，只要它包含一个有效的退订链接。[4] 尽管大多数专家都建议不要点击退订链接，因为那只会告诉更无良的垃圾邮件发送者你是一个活跃目标。国会怎么指望邮件接收者能分辨哪些退订链接会减少垃圾邮件，哪些反而会增加呢？

过滤器将有助于改善这种情况，让“选择加入”垃圾邮件发送者无法生存。这类公司无法采取严重的欺骗过滤器措施（例如伪造邮件头），同时还能维持他们列表上数千万人是“订阅者”、真正想收到他们宝贵优惠的假象。[5]

结果就是“选择加入”垃圾邮件非常容易过滤。我无法想象任何一个贝叶斯过滤器——无论实现得多糟糕——会漏掉这类邮件。我不认为有任何一封这样的邮件通过过我的过滤器。

随着垃圾邮件数量的增长促使过滤器被广泛使用，看到“选择加入”邮件营销者发送的垃圾邮件的人数将逐渐减少到零。而这些公司本身也会随之消失。

诚然，这会让错误的那一半垃圾邮件发送者关门。但除掉“选择加入”垃圾邮件发送者最终也会伤害到底层发送者。

如果“选择加入”垃圾邮件发送者消失了，在 L. L. Bean 和 Alan Ralsky 之间留下清晰的界限，那么通过区分他们的法律就会容易得多。每个人都会清楚营销在哪里结束、犯罪从哪里开始，也不会有游说者试图模糊这种界限。

在你门上装一把锁可能无法挡住所有人，但它能让你轻松区分你邀请的人和破门而入的人。
