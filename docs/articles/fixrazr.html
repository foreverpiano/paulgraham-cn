一个具体的想法

阅读原文

我刚收到一封来自 Y Combinator 合伙人的邮件。所以如果你擅长这类事情，并希望在下一轮融资中提出以此为题的工作，那么你被接受的机会会很大。

我刚拿到一部 Motorola RAZR，这是一款相当不错的手机，功能和其他小型手机大致相当。它有网页/WAP 浏览器、Java 虚拟机，能收发短信和（勉强算）电子邮件，还有一块 170x220 像素的彩色屏幕。每个功能单独用都还不错，但整体体验就像 DOS，或者是多任务 Finder 出现之前的 Mac。也就是说，要发一条 Yahoo Messenger 消息，我就得费力地选中并启动一个 Java 应用，这个应用基本会接管整个手机，等它建立网络连接等等，然后才能发送消息。只要我一直跑着这个 Java 应用，什么都不做，也许还能接收 Yahoo Messenger 消息。如果我想浏览网页，甚至只是打个电话，就得退出 ymessenger 应用，切换到其他模式。另外这部手机完全没有利用“互联网始终在线”的优势，比如它不知道如何同步日历、通讯录等。我五年前用的那部手机和现在相比，进步微乎其微。

所以我希望能看到一个 WFP 提案，做出一个不那么糟糕的手机操作系统或界面。这并非不可能：现在已经有了一些半开放的手机平台，可以运行 Linux。
