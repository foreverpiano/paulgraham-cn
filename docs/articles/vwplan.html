Viaweb 的最初商业计划

阅读原文

我最近找到了我认为是 Viaweb 最早的商业计划。当时我并不清楚一份商业计划应该包含哪些内容，所以只是把我觉得重要的东西写了下来（用纯文本格式，以便通过 email 轻松发送）。

公司最初名叫“Webgen”。我们以为只需要 15,000 美元的启动资金，后来证明这个估计过于保守。

读这份计划书，你会发现即使是我们，当时也还在努力理解什么是基于 Web 的应用。在系统描述中，我们把站点生成器和 Web 界面列为两个独立的组件；现在人们会直接把它们看作一个基于 Web 的站点生成器。我们当时还认为，有些用户会更喜欢通过 email 而不是浏览器来更新他们的网站，因为使用 Web 浏览器在当时算是一项相当冷门的技能。

MarketplaceMCI 是当时电子商务领域的巨头。他们采用服务局模式，却能做出外观精美的网站。我们最初的页面设计就是模仿他们的。

Webgen 商业计划草稿。1995 年 8 月 24 日。

概述

Webgen 允许客户远程生成 Web 产品目录。

目前有几家其他公司也能生成并提供 Web 目录服务。其中最知名的是 Marketplace MCI（www.internetmci.com）。但这些网站都是在内部使用自己的生成器，根据客户提供的材料来构建目录；而 Webgen 的生成器可以由任何拥有调制解调器和 Netscape 浏览器的人交互式地使用。

这将把上线一个产品目录的成本降低大约 90%。同时也意味着我们的业务规模不会受限于我们能亲自服务的客户数量。我们将是一家软件公司，而不是服务公司。

任何人都可以注册，在我们的网站上托管一个目录，只需支付固定的月费。而且我们把 Webgen 设计得非常易用，任何平面设计师或“Web 顾问”都能用它来生成目录。因此，我们不是去主动联系目录公司提出帮他们把目录放到网上，而是让他们自己来完成。

假设你经营一家目录公司。今天，如果你想拥有一个可以接受在线订购的目录，你有两个选择。你可以让 MCI 这样的公司为你创建和托管它，或者自己搭建服务器。两种方案都很昂贵。当我们的服务正式上线后，你就可以在公司内部自己完成：任何会用 Netscape 的人都可以在 Webgen 上生成目录。

有大量公司希望能在 Web 上销售产品，前提是成本更低且操作更简单。据我们所知，Webgen 是唯一能够挖掘这个市场的系统。

随着市场成熟，我们会考虑将独立的目录生成器/服务器作为商业产品独立出来。

Webgen 的规模有多大？

即使假设它能成功，Webgen 的前景如何？像 Webgen 这样的公司在互联网的未来会扮演多么重要的角色？

答案可能出人意料：一个非常重要的角色。技术人员和商业人士都认同，Web 将变得非常重要。而 Web 的重要性很大一部分将来自在线商务。相应地，Webgen 可能又会参与到在线商务的很大一部分当中。

为什么？暂时忘记 Webgen，单纯问问：在线商务的未来会是什么样子？当有人想在 Web 上销售商品时，他们会怎么做？目前，他们要么去找 MCI 这样的公司，要么自己搭建服务器。但一两年后他们会怎么做？十年后呢？

如果在线销售像其他软件应用一样，那么这个市场的未来将是通用软件包。你最终可以买到一份 Microsoft Catalog，然后用它把自己的目录放到网上。

有几个原因导致这种情况短期内不太可能发生。主要问题是，在线目录不像文字处理那样只发生在你自己的电脑上。它必须位于 Web 服务器上，需要加密交易、高带宽网络连接、100% 的正常运行时间、安全性、备份、订单跟踪等等。搭建服务器既昂贵又困难。在东海岸，单是网络连接一项每月就要花费约 2000 美元。

所以次优方案是使用通用的目录生成器，但让别人来负责服务器部分。而这恰恰就是 Webgen。Webgen 就是 Microsoft Catalog，或者说是最接近它的产品。不要被“软件运行在我们的机器上”这个事实误导。在网络时代，程序是运行在你的电脑上还是别处已经不再重要。使用 Webgen 将和使用任何现成软件包一样，只是用户不需要安装任何东西。

更重要的是，我们的软件将在所有人的机器上“运行”，更新免费且即时，界面也是用户已经熟悉的（Netscape）。

Webgen 的理念很像当年 Kodak 的理念。在 Kodak 出现之前，如果你拍了照片，就必须自己冲洗。如果你是一个普通人想拍照片，你得去摄影工作室，就像今天你可能会去找 MCI 把目录放到网上一样。

伊士曼卖出了一款人人都会用的相机，同时承诺处理那些繁琐且需要大量资本的部分——冲洗胶卷。

我们卖的是一款人人都会用的目录生成器，同时承诺处理那些繁琐且需要大量资本的部分——托管页面。

当然，真正高端的客户会想要自己搭建服务器，就像高端摄影客户会自己冲洗照片一样。我们不排除为他们搭建 Webgen 的克隆版本。但我们相信，如果把自己定位为销售产品而非提供服务，我们会更成功。Webgen 可以成为在线商务领域的 Kodak。总会有人成为那个 Kodak 的。

系统

Webgen 的方案包含三个组成部分：

1. 一台服务器（www.webgen.com），能够托管目录并支持订单处理。

2. 一个生成器，能够生成将在站点上托管的目录。

3. 一个界面，允许位于 Web 任何位置的用户交互式地修改并重新生成他们的目录。

网站已经上线并运行，虽然目前服务器上只有一个我们为演示制作的目录。所有三个组成部分都已经可以工作。（演示目录可以通过我们主页上的“Clients”链接查看，主页地址是 http://www.webgen.com）

我们目前正在完善软件，使其更快、更易用，但我们相信它已经是目前最先进的 Web 目录生成器。Webgen 能够做到几项我们认为其他生成器都做不到的事情：

1. 最大的优势当然是交互式界面。正因为如此，Webgen 才能成为一家把生成器卖给所有人的软件公司，而不是在内部使用它的服务公司。

但即使与 MCI 等公司在内部使用的生成器相比，我们也认为 Webgen 更为先进：

2. Webgen 会自动生成站点中的所有按钮。复杂的网站会使用图像按钮而不是文本链接。通常这些按钮图像都是在 Photoshop 之类的程序中手工制作的。我们相信我们的生成器是第一个能自动创建站点中所有按钮的。

这对用户来说是一大优势，因为这意味着他们甚至可以通过我们的远程界面来控制站点中按钮的外观。例如，如果他们希望按钮中的文字使用 Times 而不是 Helvetica（默认字体），他们只需在我们的远程界面上做一个菜单选择。下次查看网站时，所有按钮文字都会变成 Times 字体。

3. Webgen 会自动创建所有缩略图。用户只需提供目录中商品的原始（大尺寸）图像。因为缩略图是由软件生成的，它们可以全部保持相同高度，从而得到一套更一致的页面。（在这方面可以对比我们的演示和现有的在线目录。）

4. Webgen 提供了多种页面风格。例如，我们默认的分类页面风格是在每页横向排列三个缩略图。但我们已经准备了另外六种可能的分类页面风格。通过菜单选择，客户可以将分类页面布局成单个图片加编号条目、纵向排列缩略图，甚至斜向排列。对于单个商品页面和首页，也有数量相当的布局选项。

5. Webgen 会为每个目录建立索引，并创建一个数据库，由运行在我们服务器上的搜索脚本使用。用户可以通过查看索引或按关键词搜索，在我们网站上的任何目录中找到商品。

6. 除了交互式界面，Webgen 还有一套命令语言用于批量更新目录。命令可以通过 email 发送到服务器，目录会自动重新生成。这对希望将目录“连接”到库存控制系统的客户非常有用。因为重新生成一个大型目录最多只需要一两分钟，客户可以让他们的在线目录反映库存的每分钟变化——就像给我们发一封 email 一样简单，而且无需额外费用。

用户的使用方式

我们预计会有两类用户：

（A）有时用户就是目录公司本身。Webgen 非常易用，目录公司完全可以指定内部人员来管理他们的在线目录。

（B）否则用户将是不断壮大的“Web 顾问”队伍中的一员，代表目录公司行事。顾问会把我们收取的固定费用转给客户并加上自己的加价。我们预计这类增值代理商（VARs）会积极向目录公司推销这项服务。有了 Webgen，任何 Web 顾问都可以去找目录公司，提出以远低于当前市场价格的价格把目录放到网上，同时自己还能获得可观的加价。

无论哪种情况，使用 Webgen 的流程都是一样的：

1. 客户联系我们并开设新账户。我们会分配给他们一个用户名和密码，如果他们需要，我们还会注册一个新的域名。我们网站上的每个目录都可以拥有自己的直接 URL，格式为

http://www.yourcompany.com

如果客户希望如此。

2. 要生成目录，Webgen 需要两样东西：目录中包含的商品图像，以及包含价格、货号、描述等信息的文本。

用户有几种方式可以将这些原始材料发送给我们。他们可以直接在我们的远程界面中输入文本，也可以用预先指定的格式通过 email 发送到我们的服务器。

他们可以将图像登录到我们在服务器上的账户，通过 FTP 上传到指定目录，生成器会自动从中获取。（任何能支持 Netscape 的连接也能支持 FTP。）当浏览器支持带附件的图像邮件时（他们承诺很快就会实现），向 Webgen 发送图像就会变得非常简单。

3. Webgen 的服务器通过远程界面或 email 接收输入后，会自动重新生成目录中被修改的部分。用户通过 Netscape 通常在几秒钟内就能看到这些变化。

4. 当用户完成并检查完对目录的修改后，他或她会向服务器发送一条“commit”消息，使修改后的目录被复制到其实际的 URL 上。此时目录就会对访问网站的消费者可见。

出于安全考虑，commit 命令将使用一次性密码。这样，即使有人拿到了用户的普通密码，也无法修改实际在网站上显示的目录。

5. 订单量大的客户将获得我们在服务器上的一个账户，用于检索订单。如果他们愿意，可以让这个连接 24 小时保持打开状态，生成实时的订单数据流。

较小的客户会通过传真接收转发的订单——这是一个有吸引力的方案，因为它安全且使用熟悉的技术。

细节

固定费用将包含每月一定数量的网络带宽（即访客）。超过后可能收取使用费。

Netscape 最近宣布将支持 Java。这最终可能让我们把部分生成器运算转移到客户的 CPU 上。

团队

目前，Webgen 由以下四人组成：

Paul Graham（pg@webgen.com）是动态语言专家。他是《On Lisp》（Prentice Hall，1993）和《ANSI Common Lisp》（Prentice Hall，1995）的作者。他曾为美国能源部、杜邦（为他们编写了首批参数化 CAD 程序之一）和 Interleaf 担任顾问。他以 summa cum laude 的成绩从康奈尔大学获得文学学士学位，并在哈佛大学获得计算机科学博士学位。

Gino Lee（gino@webgen.com）是哈佛大学印刷与排版导师，同时担任哈佛实验排版工坊 Bow & Arrow Press 的主任；他教授包括凸版印刷、设计史、数字字体设计和艺术家书籍等科目。自 1989 年以来，他一直独立从事数字字体和平面设计工作。1990 年他获得 Society of Printers 卓越奖。

Robert Morris（rtm@webgen.com）是计算机网络专家。他发表过关于高速网络、无线通信和计算机安全的论文。还在高中时，他就在贝尔实验室工作，在那里将 TCP/IP（互联网所基于的软件）移植到贝尔实验室版本的 Unix 上。后来他为 Convex Computers、Thinking Machines 和 DEC Systems Research Center 工作。他拥有哈佛大学计算机科学文学学士学位，目前是该校计算机科学博士候选人。

Julian Weber（jlw@webgen.com）在纽约从事法律工作超过 30 年。从 1964 年到 1979 年，他是 Botein, Hays, and Sklar 律所的合伙人。从 1979 年到 1984 年，他担任《The National Lampoon》的总裁。他拥有哈佛法学院的法学博士学位。

Robert 和 Paul 负责软件；Gino 主管平面设计；Julian 处理公司和法律事务。我们正在寻找第五个人来管理业务事务，特别是与媒体、客户和潜在企业合作伙伴打交道。

财务

Webgen 目前没有资金。另一方面，它目前的运行成本非常低。我们所有的设备都是团队成员的个人财产，业务没有其他主要开支。没有人领薪水。

如果有必要，公司可以实现自我融资。也就是说，我们可以用初期客户的收入来购买能帮助我们吸引更多客户的设备。

如果我们有大约 15,000 美元可供花费，业务将会增长得快得多。这笔钱可以用来：

1. 购买一台用于开发软件的电脑（3000 美元）。

2. 购买安全的服务器软件（5000 美元）。这似乎不是绝对必要的；网上有很多网站允许你直接发送未加密的信用卡号，至今还没有信用卡号被盗的报道。但目录公司可能会认为安全的连接是必要的，而这 5000 美元的投入会让 Webgen 显得更专业。

3. 高带宽的互联网连接（3000 美元 + 每月 350 美元）。目前 Webgen 只有 28.8kb 的连接。这在初期客户很少的时候还能应付。每月 350 美元我们可以把服务器放在 ISP 的机房，使用他们 T1 线路的一部分。这也意味着需要再买一台服务器，又要花费 3000 美元。

花些钱做广告可能也是个好主意。
